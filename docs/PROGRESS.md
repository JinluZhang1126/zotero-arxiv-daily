# 变更记录

## 2026-07-14 23:45 CST - 在论文列表页面标明数据源

- 遇到的问题：论文信息实际保存在仓库根目录 `paper_list.json`，但 `index.html` 没有说明数据来源，用户无法从页面确认或直接查看原始数据。
- 修改原因：让页面明确关联当前的结构化论文数据，同时确保定时任务重新生成页面后仍保留该信息。
- 修改内容：在 `index.html` 和 `main.py` 的 HTML 生成模板中增加指向 `./paper_list.json` 的数据源链接。
- 解决与验证：确认 `main.py` 的保存流程先写入 `paper_list.json` 再生成 `index.html`；使用系统 Python 隔离执行渲染函数，验证空列表、示例论文、HTML 转义和已提交页面一致；运行 `py_compile`、JSON 格式检查和 `git diff --check`。本机未安装 `uv`，因此未使用 `uv run`。
- 后续避免：调整生成产物时必须同时修改生成模板与已提交产物，并验证二者一致，避免下次工作流运行覆盖手工改动。
- Git commit ID：`83dfbe8`

## 2026-07-15 00:01 CST - 按日期累计论文历史

- 遇到的问题：`paper_list.json` 每次运行都以写入模式整体覆盖，`index.html` 因此只能展示最近一次任务的论文，无法查看历史推荐。
- 修改原因：保留不同运行日期的论文，并让页面按日期从新到旧累计展示。
- 修改内容：为每篇论文增加 `date` 字段；保存前读取已有 JSON，同日重跑时替换当天批次，重复 arXiv ID 只保留最新记录，其余历史继续保留并按日期倒序排列；页面增加日期列；README 同步说明累计行为。
- 解决与验证：使用临时 JSON 隔离测试首次写入、跨日追加、同日替换、跨日去重、倒序排列、页面总数与 HTML 转义；运行 `py_compile`、JSON 格式检查和 `git diff --check`。测试过程中发现系统 Python 不支持运行时求值 `str | None`，改用兼容性更好的 `Optional[str]`。
- 后续避免：涉及历史数据的写入逻辑必须覆盖首次运行、重复运行、跨日运行和重复论文四种场景，不能只验证单次生成结果。
- Git commit ID：`4b41d2e`
